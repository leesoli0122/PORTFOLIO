> 배포 URL: https://leesoli0122.github.io/PORTFOLIO/POTATO/
> 마지막 업데이트: 2026-04

---

## 프로젝트 개요

- **목적**: 프로크리에이트로 그린 캐릭터 파츠를 업로드하고 조합해 PNG 이모티콘으로 저장하는 개인용 웹 앱
- **사용 환경**: iPad Safari + PC 브라우저
- **기술 스택**: HTML / CSS / JavaScript
- **이미지 저장 방식**: IndexedDB 사용, 서버 저장 없음
- **배포 방식**: GitHub Pages

이 프로젝트는 제가 직접 쓰기 위해 만든 개인 작업 도구입니다.  
파츠 이미지를 서버에 올리지 않고 브라우저 안에 저장한 뒤, 얼굴/귀/옷/포즈/배경을 조합해서 PNG로 내려받을 수 있게 구성했습니다.

---

## 작업 범위와 AI 활용

이 프로젝트는 제 스스로의 힘만으로 완성한 작업이라고 말하기 어렵습니다.  
화면 구조, 페이지 흐름, CSS 분리 방식, 버튼/카드/탭 같은 UI 정리는 제가 직접 방향을 잡고 수정했습니다. 다만 JavaScript의 복잡한 기능 구현은 AI의 도움을 많이 참고했습니다.

### 직접 작업한 부분

- 메인, 업로드, 메이커, 갤러리 화면 구성
- CSS 파일 구조 분리
- 버튼, 탭, 카드, 모달, 토스트 UI 스타일 정리
- 페이지별 사용 흐름 정리
- 프로젝트 목적과 필요한 기능 정리
- 실제 화면을 보면서 여백, 크기, 문구, 배치 수정

### AI 도움을 받아 구현하거나 정리한 부분

- IndexedDB 저장 구조
- 이미지 업로드 후 base64 dataURL로 저장하는 흐름
- 캔버스에 파츠를 레이어 순서대로 그리는 기능
- PNG 저장 및 다운로드 기능
- 즐겨찾기, 검색/필터 성격의 상태 관리
- 갤러리 카테고리와 이름 수정 기능
- 일부 JavaScript 오류 수정과 코드 구조 정리

제가 과장해서 "전부 직접 개발했다"고 말하기보다, 퍼블리셔로서 화면 구조와 UI를 만들고 JavaScript 기능은 AI 도움을 받아 이해하며 적용한 프로젝트라고 정리하는 게 맞습니다.

## 현재 구현된 기능

### 1. 메인 페이지

- 캐릭터 만들기, 파츠 업로드, 갤러리로 이동하는 진입 화면
- 큰 버튼형 메뉴로 주요 기능을 바로 선택할 수 있게 구성
- 간단한 배경 데코와 로고 애니메이션 적용

### 2. 파츠 업로드

- 얼굴, 귀, 옷, 포즈, 배경 탭 구성
- PNG / JPEG / WebP 이미지 업로드
- 여러 장 동시 업로드 가능
- 업로드한 이미지를 IndexedDB `parts` 스토어에 저장
- 등록된 파츠 개수 표시
- 최신순 / 이름순 정렬
- 파츠 이름 인라인 수정
- 파츠 삭제

### 3. 캐릭터 만들기

- 탭별 파츠 선택
- 선택한 파츠를 캔버스에 실시간 반영
- 레이어 순서대로 이미지 합성
- 즐겨찾기 등록/해제
- 즐겨찾기만 보기
- 랜덤 조합
- 초기화
- 카카오 사이즈 PNG 저장
- OGQ 사이즈 PNG 저장
- 저장 시 갤러리 DB에도 기록

### 4. 갤러리

- 저장된 완성작 목록 표시
- 카드 클릭 시 상세 모달 표시
- PNG 다운로드
- 완성작 삭제
- 완성작 이름 수정
- 카테고리 생성
- 전체 / 미분류 / 사용자 생성 카테고리 탭 필터
- 완성작 카테고리 변경
- 카테고리 삭제 시 해당 완성작을 미분류로 이동

## 레이어 순서

캔버스에서는 아래 순서대로 이미지를 겹쳐 그립니다.

```text
배경(background) → 포즈(pose) → 옷(outfit) → 얼굴(face) → 귀(ear)
```

이 순서가 중요한 이유는 이미지가 나중에 그려질수록 위에 올라오기 때문입니다. 예를 들어 얼굴보다 배경이 뒤에 있어야 하므로 배경을 먼저 그리고, 귀는 가장 위에 보여야 해서 마지막에 그립니다.

## 아직 보완할 점

- 업로드 페이지의 파츠 순서 직접 변경 기능은 아직 완성되지 않았습니다. 현재는 최신순 / 이름순 정렬만 됩니다.
- 갤러리에는 저장 직후 이름 입력 모달 구조가 있지만, 메이커 저장 흐름과의 연결은 다시 점검이 필요합니다.
- IndexedDB와 Canvas 관련 코드는 AI 도움을 받아 구현했기 때문에, 추후 직접 설명할 수 있도록 주석과 문서를 더 보강할 예정입니다.
- iPad Safari에서 이미지 업로드, 저장, 다운로드 흐름을 한 번 더 꼼꼼히 확인해야 합니다.
- 현재는 개인용 정적 웹 앱 기준이라 서버 저장, 계정, 동기화 기능은 없습니다.

---

## 배운 점

- 화면을 먼저 나누고 CSS를 공통/페이지 단위로 분리하면 유지보수가 편해진다는 점을 경험했습니다.
- 브라우저 안에서도 IndexedDB를 이용하면 서버 없이 이미지를 저장할 수 있다는 걸 알게 되었습니다.
- Canvas는 단순히 그림을 그리는 용도뿐 아니라 여러 이미지를 합성해 새 PNG로 저장하는 데 사용할 수 있다는 걸 배웠습니다.
- JavaScript 기능을 AI 도움으로 구현했더라도, 그대로 붙여넣기보다 파일 역할과 데이터 흐름을 나눠서 이해하는 과정이 중요하다는 걸 느꼈습니다.

퍼블리싱 구조와 UI 정리는 직접 진행했고, IndexedDB와 Canvas 중심의 JavaScript 기능은 AI 도움을 받아 구현하며 학습했습니다.
