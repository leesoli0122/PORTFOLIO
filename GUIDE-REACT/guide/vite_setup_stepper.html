GUIDE-REACT Vite 프로젝트 셋업 단계별 가이드

터미널에 순서대로 입력하세요
# 1. Vite + React 프로젝트 생성 npm create vite@latest GUIDE-REACT -- --template react # 2. 프로젝트 폴더로 이동 cd GUIDE-REACT # 3. 기본 패키지 설치 npm install # 4. SCSS 설치 (퍼블리셔 필수) npm install -D sass # 5. 라우팅 라이브러리 설치 npm install react-router-dom # 6. 개발 서버 실행 확인 npm run dev
브라우저에서 http://localhost:5173 열리면 설치 성공이에요.
Node.js가 없으면 먼저 nodejs.org에서 LTS 버전을 설치하세요. 터미널에서 node -v로 확인할 수 있어요.
기본 생성된 파일 정리 후 최종 구조
GUIDE-REACT/ ├── index.html ← 건드리지 않아도 돼요 ├── vite.config.js ← 경로 alias 설정할 파일 ├── package.json └── src/ ├── main.jsx ← React 앱 시작점 ├── App.jsx ← 라우팅 설정 (ia.html 역할) ├── pages/ ← views/ 역할 │ ├── MainPage.jsx │ └── ComponentPage.jsx ← 컴포넌트 쇼케이스 ├── components/ ← UI 컴포넌트 모음 │ └── Button/ │ ├── Button.jsx │ ├── Button.scss │ └── index.js ├── hooks/ ← 커스텀 훅 모음 └── styles/ ← 전역 SCSS ├── main.scss ← 전역 스타일 진입점 ├── _variables.scss ← 색상·간격 변수 ├── _reset.scss ├── _typography.scss └── _mixins.scss ← 반응형 등 반복 패턴
Vite 생성 직후 src/App.css, src/index.css, src/assets/react.svg는 삭제하세요. 우리가 직접 만들 거예요.
vite.config.js — 경로 alias 설정
왜 필요한가요?
vite.config.js
import { defineConfig } from 'vite' import react from '@vitejs/plugin-react' import path from 'path' export default defineConfig({ plugins: [react()], resolve: { alias: { // '@/components/Button' 이렇게 쓸 수 있어요 // '../../../components/Button' 이런 지옥 탈출 '@': path.resolve(__dirname, './src') } }, css: { preprocessorOptions: { scss: { // 모든 scss 파일에 변수·믹스인 자동 주입 additionalData: `@use '@/styles/variables' as *; @use '@/styles/mixins' as *;` } } } })
이제 어디서든 import Button from '@/components/Button' 이렇게 쓸 수 있어요. ../../.. 지옥이 끝납니다.

라우팅 설정 — main.jsx + App.jsx
src/main.jsx
import React from 'react' import ReactDOM from 'react-dom/client' import { BrowserRouter } from 'react-router-dom' import App from './App' import './styles/main.scss' ← 전역 CSS 여기서 한 번만 ReactDOM.createRoot(document.getElementById('root')).render( <BrowserRouter> <App /> </BrowserRouter> )
src/App.jsx — ia.html 역할
import { Routes, Route } from 'react-router-dom' import MainPage from '@/pages/MainPage' import ComponentPage from '@/pages/ComponentPage' function App() { return ( <Routes> <Route path="/" element={<MainPage />} /> <Route path="/components" element={<ComponentPage />} /> </Routes> ) } export default App
퍼블의 ia.html처럼 App.jsx에서 전체 페이지 구조를 한눈에 파악할 수 있어요.
