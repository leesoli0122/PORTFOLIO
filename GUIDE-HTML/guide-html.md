# 퍼블리싱 가이드 (GUIDE-HTML)

> 배포 URL: https://leesoli0122.github.io/PORTFOLIO/GUIDE-HTML/views/
> 문서 목적: 공통 UI 컴포넌트와 퍼블리싱 가이드 화면을 정리한 프로젝트 문서

---

## 프로젝트 개요

- **목적**: 자주 사용하는 UI 컴포넌트의 마크업, 스타일, 상태를 정리하는 퍼블리싱 가이드
- **사용 환경**: PC 브라우저 기준 가이드 화면
- **기술 스택**: HTML / CSS / SCSS / JavaScript / jQuery
- **외부 스크립트**: 프로젝트 내부에 포함된 jQuery, jQuery UI, Swiper 파일 사용
- **주요 사용 상황**: 버튼, 체크박스, 배지, 알럿 같은 공통 컴포넌트 구조 확인

이 프로젝트는 공통 UI를 한곳에 모아 확인하기 위한 퍼블리싱 가이드입니다.  
컴포넌트별 HTML 구조와 CSS/SCSS 작성 방식을 연습하고, 상태값을 표시하는 가이드 화면을 구성했습니다.

---

## 작업 범위와 AI 활용

컴포넌트 마크업, SCSS/CSS 구조, 가이드 화면의 기본 레이아웃은 직접 작성하고 수정했습니다.  
컴포넌트 목록을 클릭해 예시를 불러오는 흐름, 상태 카운트 갱신, 일부 jQuery 동작은 AI를 참고해 구조를 잡았습니다.

### 직접 작업한 부분

- Button, Checkbox, Badge, Alert 등 컴포넌트 예시 마크업 작성
- 컴포넌트 목록과 가이드 화면 레이아웃 구성
- SCSS와 CSS 파일 구조 정리
- 컴포넌트별 클래스 네이밍과 상태 클래스 정리
- 실제 프로젝트에서 반복 사용하기 쉬운 UI 패턴 정리

### AI 도움을 받아 구현하거나 정리한 부분

- 컴포넌트 상태 클래스에 따라 완료/진행중/대기 카운트를 계산하는 로직
- 가이드 화면에서 컴포넌트 예시를 불러오는 흐름
- preview tab 같은 일부 가이드 전용 인터랙션
- jQuery 기반 DOM 탐색과 이벤트 처리 구조 정리

## 폴더 구조

```text
GUIDE-HTML/
├── guide-html.md               # 프로젝트 정리 문서
├── views/
│   ├── index.html              # 컴포넌트 가이드 메인
│   ├── layout.html             # 레이아웃 테스트 화면
│   └── guide/
│       ├── input/              # 입력 관련 컴포넌트
│       ├── feedback/           # 알림/상태 컴포넌트
│       ├── data/               # 데이터 표시 컴포넌트
│       └── surfaces/           # 아코디언 등 표면 컴포넌트
└── assets/
    ├── css/                    # 컴파일된 CSS 및 공통 스타일
    ├── js/                     # 가이드 동작 스크립트
    └── scss/                   # 컴포넌트별 SCSS 원본
```

## 현재 구현된 기능

### 1. 컴포넌트 가이드 메인

- 좌측 컴포넌트 목록 구성
- 우측 컴포넌트 예시 표시 영역 구성
- 전체/완료/진행중/대기 상태 카운트 표시
- 컴포넌트 상태 클래스 기준으로 상태 뱃지 출력

### 2. 입력 컴포넌트

- Autocomplete
- Button
- Button Group
- Checkbox

### 3. 피드백 컴포넌트

- Alert
- Badge

### 4. 데이터/표면 컴포넌트

- Aspect Ratio
- Accordion

## 아직 보완할 점

- 컴포넌트 파일명 대소문자가 섞여 있어 배포 경로에서 링크 점검이 필요합니다.
- 일부 컴포넌트는 예시만 있고 상세 사용 규칙이나 상태별 설명이 부족합니다.
- SCSS 원본과 컴파일된 CSS의 반영 상태를 계속 맞춰야 합니다.
- 가이드용 스크립트는 AI 도움을 받아 정리했기 때문에, jQuery 이벤트 흐름을 다시 읽고 설명할 수 있게 문서화할 예정입니다.
- 실제 금융권 관리자 시스템에 가져다 쓰려면 버튼, 폼, 그리드 주변 컴포넌트를 더 촘촘히 보강해야 합니다.

---

## 배운 점

- 컴포넌트를 페이지 안에서 흩어 작성하는 것보다 가이드로 모아두면 반복 UI를 관리하기 쉽다는 걸 배웠습니다.
- 상태 클래스만 바꿔도 가이드 현황이 자동으로 바뀌게 만들면 작업 진행률을 확인하기 편합니다.
- jQuery 기반 프로젝트에서는 DOM 구조와 클래스 네이밍을 안정적으로 잡아두는 것이 스크립트 유지보수에 중요하다는 걸 느꼈습니다.

---

## 한 줄 소개

반복해서 사용하는 UI 컴포넌트의 마크업과 스타일을 정리한 퍼블리싱 가이드입니다.  
컴포넌트 구조와 CSS/SCSS 정리는 직접 진행했고, 상태 카운트와 일부 jQuery 동작은 AI 도움을 받아 구현하며 학습했습니다.
